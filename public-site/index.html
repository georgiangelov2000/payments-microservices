Payment orchestration

Route, normalize, and observe provider workflows.

The gateway makes provider selection, request handling, and response normalization a platform concern instead of merchant-side code.

1. AuthenticateRedis gateway profile lookup by hashed API key.
2. AuthorizeRoute, provider, merchant, and subscription checks.
3. RouteSend the normalized request to Stripe, PayPal, or future providers.
4. NormalizeReturn consistent status, URL, provider reference, and workflow state.

API-first architecture

Optimized for low-latency gateway traffic.

The gateway reads denormalized access profiles from Redis first, falls back to a single indexed database lookup, and avoids relational joins during normal API execution.
