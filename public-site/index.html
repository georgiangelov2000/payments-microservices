How it works

Route, authenticate, normalize, and observe — all in one request cycle.

01

Authenticate

Redis gateway profile lookup by HMAC-hashed API key. Plaintext keys are never stored.

02

Authorize

Validate route, provider, merchant status, and subscription scope — all from the cached profile.

03

Route

Forward the normalized request to the selected processor — Stripe, PayPal, or future providers.

04

Normalize

Return consistent status, checkout URL, provider reference, and workflow state regardless of processor.

Security

Designed for payment data control from day one.

HMAC-hashed API keys

Gateway authentication uses HMAC-SHA256 hashes. Plaintext keys are never persisted.

Scoped route enforcement

Allowed routes, providers, and permissions are included in gateway profiles and enforced on every request.

Payload isolation

Merchant summaries stay readable while raw provider payloads remain accessible only to authorized operator views.

Redis-cached profiles

Auth profiles are flattened and cached at gateway level. DB is never queried on the standard payment path.
