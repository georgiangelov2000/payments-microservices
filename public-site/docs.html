Developer documentation

Build payments through one orchestration API.

Use PayFlow to create payment requests, send customers to provider checkout, inspect workflow status, and normalize provider responses across processors.

Authentication

Send the merchant gateway API key with every request. The gateway hashes the key and checks a Redis-cached access profile.

X-Api-Key: pgw_test_your_key_here

Create Payment

Create a checkout session or order through the unified gateway endpoint.

POSThttp://localhost:8080/api/v1/payments
curl -X POST http://localhost:8080/api/v1/payments \
  -H "X-Api-Key: pgw_test_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "order_id": 1001,
    "amount": 1,
    "price": "49.00",
    "alias": "stripe"
  }'
await fetch("http://localhost:8080/api/v1/payments", {
  method: "POST",
  headers: {
    "X-Api-Key": "pgw_test_your_key_here",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    order_id: 1001,
    amount: 1,
    price: "49.00",
    alias: "stripe"
  })
})
$payload = json_encode([
  'order_id' => 1001,
  'amount' => 1,
  'price' => '49.00',
  'alias' => 'paypal',
]);

$ch = curl_init('http://localhost:8080/api/v1/payments');
curl_setopt_array($ch, [
  CURLOPT_POST => true,
  CURLOPT_HTTPHEADER => [
    'X-Api-Key: pgw_test_your_key_here',
    'Content-Type: application/json',
  ],
  CURLOPT_POSTFIELDS => $payload,
  CURLOPT_RETURNTRANSFER => true,
]);
Http::withHeaders([
    'X-Api-Key' => config('services.payflow.key'),
])->post('http://localhost:8080/api/v1/payments', [
    'order_id' => 1001,
    'amount' => 1,
    'price' => '49.00',
    'alias' => 'stripe',
]);
import fetch from "node-fetch";

const response = await fetch("http://localhost:8080/api/v1/payments", {

Payment Status

Use status and tracking endpoints to inspect provider workflow events and normalized payment states.

GET/api/v1/payments/{id}/show
GET/api/v1/payments/{id}/tracking

Refunds

Refund endpoint placeholder for provider-specific refund orchestration.

POST /api/v1/refunds

Tokenization

Future-ready vault/token flows can be exposed without merchant-side provider rewrites.

POST /api/v1/tokens

Routing

Routing policies can consider provider availability, merchant configuration, market, currency, cost, and fallback rules.

{
  "provider_preference": ["stripe", "paypal"],
  "fallback_enabled": true,
  "currency": "USD",
  "country": "US"
}
